KEY FINDINGS
============

CPU & Clocking:
  - Motorola 68000 @ 8 MHz (÷2 from 16 MHz master clock)
  - Z80 @ 4 MHz (sound CPU)
  - No tilemap hardware (sprite-only architecture)

Graphics:
  - X1-001A + X1-002A sprite generators
  - 16×16 fixed-size sprites, 4 bpp (16 colors)
  - 256 max active sprites per frame
  - 384×240 display @ 60 Hz, 15-bit RGB palette (2048 colors)
  - Sprite ROM: 1–4 MB per game (likely uncompressed)

Audio:
  - YM2610 OPNB (Superman, Gigandes, Balloon Brothers, etc.)
  - YM2151 OPM (Twin Hawk only)
  - Both sound chips require instantiation

Games:
  - 7 confirmed titles (Superman, Twin Hawk/Daisenpu, Last Striker, Gigandes, Balloon Brothers, Blue Shark, Don Doko Don)
  - Genres: Shooters (4), beat 'em up (1), puzzle (1), sports (1)
  - Regional variants (Superman 3, Gigandes 2)

Implementation Complexity:
  - MODERATE (lower than Taito B/Z/F3 due to sprite-only, no tilemaps)
  - Estimated effort: 110 hours (Phase 1: 20h, Phase 2: 30h, Phase 3: 25h, Phase 4: 35h)
  - No existing FPGA implementations to reference
  - MAME source is mature reference implementation

Unknowns Requiring Reverse-Engineering:
  - X1-001A sprite ROM addressing formula (sprite_code → ROM offset)
  - Y-coordinate attribute RAM exact layout (0xD00000)
  - Sprite rendering order & priority
  - Color palette application (5-bit field mapping)
  - CPU interrupt timing & vblank detection
  - 68000 ↔ Z80 communication protocol
