KEY RESEARCH FINDINGS
================================================================================

REGISTER MAP (Complete, 16 words):
- Word 0x00-0x07: Layer scroll (SCROLL0-3 X/Y)
- Word 0x08: Rowscroll control
- Word 0x09: LAYER_CTRL (priority, layer enable)
- Word 0x0A: SPRITE_CTRL (size, sort mode)
- Word 0x0B: LAYER_SIZE (tilemap dimensions)
- Word 0x0C: COLOR_KEY (transparent color)
- Word 0x0D: BLEND_CTRL (blending mode)
- Word 0x0E: STATUS (chip status, read-only)
- Word 0x0F: (reserved)

SPRITE SYSTEM (256 total):
- Format: 4 words per sprite (code, color, X/Y/size, priority/blend)
- Sizes: 16×16, 32×32, 64×64, 128×128 pixels (composite from ROM tiles)
- Color: 6-bit palette bank (16 colors per bank)
- Priority: bit controls sprite vs foreground layer ordering
- List: Parsed during VBLANK, rendered per-scanline during visible period

BACKGROUND LAYERS (2-4 configurable):
- Tiles: 16×16 pixels, 4bpp, from character ROM
- Scroll: Global X/Y per layer, optional per-row offset (rowscroll)
- Priority: Ordering defined by LAYER_CTRL register bits
- Transparent: Color 0 in palette

RENDERING (8 MHz pixel clock):
- Sprite scanner: Evaluates all 256 sprites, builds active list per scanline
- BG renderer: Per-layer tilemap walk (X/Y scroll) → character fetch → pixel
- Priority mixer: Selects final pixel based on layer priority and sprite.priority
- Output: 320×224 visible area within 384×272 total frame (76.5 Hz refresh)

CRITICAL DETAILS:
1. Sprite tile prefetch: 128×128 = 64 tiles = 8 KB — must buffer before line render
2. Register staging: CPU writes → shadow regs, transferred on VSYNC for coherency
3. ROM arbitration: Sprite fetch priority > BG fetch (interleaved)
4. Priority encoding: Sprite.priority=1 = above foreground, 0 = below
5. Rowscroll: Rarely used; basic implementation sufficient, defer advanced
